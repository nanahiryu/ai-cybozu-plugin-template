# /do-plan - プランに従って実装を行う

プランに従って実装を行うコマンド

## タスク

1. プランに従い、issue の内容を実装する
   (並列実行可能なタスクがあれば並列で実行させてください)
2. 実装が完了したらレビューを実施し、問題なければ PR を作成する

## タスクの進め方

- 親エージェントは subagent を効果的に利用してください
- 並列実行可能なタスク（変更範囲が被らず、依存関係のないタスク）があれば並列で実行させてください
- コード実装には coder subagent を利用してください
- レビューには reviewer subagent を利用してください

## break point

以下に設定した機能が動くようになったら、作業を止めてユーザーに確認を依頼してください
また、確認を依頼する場合は develop を base ブランチとする PR を作成し、そこまでの差分をその PR に全て含めるように作業を進めてください

- プラグイン設定画面が完成した時
- ハンドラが一つ完成するごと
- プラグインが完全に機能し、全ての E2E テストが通るようになった時

## テスト設計にミスがあり、修正が必要と判断したとき

- 人間に確認を依頼してください
